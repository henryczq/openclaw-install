## 一键安装

### 功能概述

一键安装功能帮助用户自动完成 OpenClaw 的完整环境搭建，包括依赖环境检测、Node.js 安装、Git 安装、OpenClaw 安装、配置初始化和网关启动。

### 操作流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  环境检测   │ -> │ 安装Node.js │ -> │  配置npm    │ -> │  安装Git    │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
                                                                  │
       ┌──────────────────────────────────────────────────────────┘
       ▼
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│安装OpenClaw │ -> │ 初始化配置  │ -> │ 启动网关    │ -> │  完成安装   │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

### 详细步骤说明

#### 步骤 1: 环境检测

**目的**：检测系统是否已安装必要的依赖环境

**检测内容**：
- **Node.js**：检查是否已安装，版本是否 >= 22.0.0
- **NPM**：检查是否已安装，版本信息
- **Git**：检查是否已安装，版本是否 >= 2.40.0
- **OpenClaw**：检查是否已安装，版本信息

**详细操作**：
1. 执行 `node --version` 命令检测 Node.js
2. 执行 `npm --version` 命令检测 NPM
3. 执行 `git --version` 命令检测 Git
4. 执行 `openclaw --version` 命令检测 OpenClaw

**结果处理**：
- 如果环境已安装且版本符合要求，跳过对应安装步骤
- 如果环境未安装或版本过低，执行自动安装

#### 步骤 2: 安装 Node.js

**目的**：安装或升级 Node.js 运行时环境

**详细操作**：
1. 从配置的镜像源下载 Node.js 安装包
   - 默认镜像：淘宝 npm 镜像
   - 默认版本：v24.14.0
   - 下载地址：`https://cdn.npmmirror.com/binaries/node/v24.14.0/node-v24.14.0-x64.msi`
2. 执行 MSI 安装程序
3. 等待安装完成并验证版本

**手动安装选项**：
- 如果自动下载失败，可点击"手动下载"按钮
- 跳转到 Node.js 官网下载页面

#### 步骤 3: 配置 NPM

**目的**：配置 npm 使用国内镜像源，加速后续包下载

**详细操作**：
1. 执行 `npm config set registry https://registry.npmmirror.com`
2. 配置完成后验证：`npm config get registry`

#### 步骤 4: 安装 Git

**目的**：安装 Git 版本控制工具

**详细操作**：
1. 从配置的镜像源下载 Git 安装包
   - 默认镜像：南京大学镜像
   - 默认版本：v2.53.0
   - 下载地址：`https://mirror.nju.edu.cn/github-release/git-for-windows/git/...`
2. 执行 EXE 安装程序
3. 等待安装完成并验证版本

**手动安装选项**：
- 如果自动下载失败，可点击"手动下载"按钮
- 跳转到 Git 官网下载页面

#### 步骤 5: 安装 OpenClaw

**目的**：安装 OpenClaw 核心程序

**详细操作**：
1. 设置 PowerShell 执行策略：`Set-ExecutionPolicy RemoteSigned -Scope CurrentUser -Force`
2. 执行官方安装脚本：`iwr -useb https://openclaw.ai/install.ps1 | iex`
3. 等待安装完成（约 1-3 分钟）
4. 验证安装：`openclaw --version`

**安装内容**：
- OpenClaw CLI 工具
- 核心依赖包
- 默认插件

#### 步骤 6: 初始化配置

**目的**：创建 OpenClaw 初始配置文件

**详细操作**：
1. 创建配置文件目录：`~/.openclaw/`
2. 创建配置文件：`openclaw.json`
3. 写入默认配置：
   ```json
   {
     "agents": {
       "list": [
         {
           "id": "main",
           "provider": "anthropic",
           "model": "claude-3-7-sonnet-20250219"
         }
       ],
       "defaults": {
         "provider": "anthropic",
         "model": "claude-3-7-sonnet-20250219"
       }
     }
   }
   ```

**配置路径**：
- 默认路径：`~/.openclaw/openclaw.json`
- 可在"设置"中修改配置路径

#### 步骤 7: 启动网关

**目的**：启动 OpenClaw 网关服务，提供 API 接口

**详细操作**：
1. 执行命令：`openclaw gateway start`
2. 后台启动网关进程
3. 循环检测端口 18789 是否开放（最多检测 30 次，每次间隔 2 秒）
4. 端口开放表示网关启动成功

**验证方式**：
- 检测本地端口 18789 状态
- 使用 PowerShell：`Test-NetConnection -ComputerName localhost -Port 18789`

### 安装日志

- 右侧显示实时安装日志
- 包含时间戳和操作详情
- 自动滚动到最新日志
- 可用于排查安装问题

## AI配置

### 功能概述

AI配置功能用于设置 OpenClaw 使用的大语言模型和 API 密钥，支持多种国产大模型。

### 操作流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ 选择模型    │ -> │ 获取API Key │ -> │ 填写API Key │ -> │ 保存配置    │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

### 详细步骤说明

#### 步骤 1: 选择 AI 模型

**支持的模型**：

| 模型 | 提供商 | 特点 |
|------|--------|------|
| 豆包 Seed 2.0 Code | 字节跳动 | 代码生成专用模型 |
| 豆包 Seed 2.0 Pro | 字节跳动 | 专业版通用模型 |
| 豆包 Seed 2.0 Lite | 字节跳动 | 轻量版快速响应 |
| 豆包 Seed Code | 字节跳动 | 代码辅助模型 |
| MiniMax M2.5 | MiniMax | MiniMax大模型 |
| GLM-4.7 | 智谱AI | 智谱AI大模型 |
| DeepSeek V3.2 | 深度求索 | 深度求索大模型 |
| Kimi K2.5 | 月之暗面 | 月之暗面大模型 |

**API 端点**：
- 默认端点：`https://ark.cn-beijing.volces.com/api/coding/v3`
- DeepSeek 端点：`https://api.deepseek.com/v1`

#### 步骤 2: 获取 API Key

**方式一：自动获取（推荐）**
1. 点击"获取 API Key"按钮
2. 应用会打开内置浏览器窗口
3. 跳转到火山引擎登录页面
4. 用户登录后，应用自动提取 API Key
5. 自动填充到输入框

**方式二：手动获取**
1. 访问火山引擎控制台：`https://console.volcengine.com/`
2. 登录账号
3. 进入"方舟"产品页面
4. 创建或查看 API Key
5. 复制 Key 并手动粘贴到输入框

#### 步骤 3: 填写 API Key

**输入要求**：
- API Key 不能为空
- 支持显示/隐藏切换（保护隐私）
- 建议保存到安全位置

#### 步骤 4: 保存配置

**详细操作**：
1. 根据选择的模型类型，调用不同的配置接口
2. **DeepSeek 模型**：
   - 构造配置对象：baseUrl、apiKey、modelId、modelName
   - 调用 `configOpenClawAI(configData, '')`
3. **其他模型（Volcengine）**：
   - 调用 `configOpenClawAI(model, apiKey)`
4. 配置写入 `openclaw.json` 文件
5. 显示保存结果

**配置生效位置**：
- 配置文件：`~/.openclaw/openclaw.json`
- 配置节点：`agents.list` 和 `agents.defaults`

## 渠道配置

### 功能概述

渠道配置功能用于配置 OpenClaw 与外部平台的集成，目前支持飞书（Feishu/Lark）渠道。

### 操作流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│安装飞书插件 │ -> │创建飞书应用 │ -> │填写应用凭证 │ -> │保存渠道配置 │ -> │重启服务     │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

### 详细步骤说明

#### 步骤 1: 安装飞书插件

**目的**：安装 OpenClaw 的飞书集成插件

**详细操作**：
1. 点击"安装飞书插件"按钮
2. 执行命令安装插件（具体命令取决于 OpenClaw 插件系统）
3. 等待安装完成
4. 验证插件安装状态

**插件功能**：
- 接收飞书消息
- 发送回复到飞书
- 处理群聊和私聊

#### 步骤 2: 创建飞书应用

**目的**：在飞书开放平台创建企业应用

**详细操作**：
1. 点击"打开飞书开发者后台"按钮
2. 浏览器跳转到：`https://open.feishu.cn/app`
3. 登录飞书账号
4. 点击"创建企业自建应用"
5. 填写应用名称和描述
6. 记录 App ID 和 App Secret

**应用权限配置**：
- 需要开启机器人能力
- 配置消息接收事件
- 设置回调地址（OpenClaw 网关地址）

#### 步骤 3: 填写应用凭证

**需要的信息**：
- **App ID**：飞书应用的唯一标识
- **App Secret**：飞书应用的密钥

**获取位置**：
- 飞书开发者后台 -> 应用详情 -> 凭证与基础信息

#### 步骤 4: 保存渠道配置

**详细操作**：
1. 验证 App ID 和 App Secret 是否填写
2. 调用 `configFeishuChannel(appId, appSecret)`
3. 配置写入 `openclaw.json` 文件
4. 显示保存结果

**配置格式**：
```json
{
  "channels": {
    "feishu": {
      "appId": "your-app-id",
      "appSecret": "your-app-secret"
    }
  }
}
```

#### 步骤 5: 重启 OpenClaw 服务

**目的**：使渠道配置生效

**详细操作**：
1. 点击"重启服务"按钮
2. 调用 `restartOpenClaw()`
3. 等待服务重启完成
4. 验证渠道连接状态

## 卸载清理

### 功能概述

卸载清理功能用于完全移除 OpenClaw 及其相关配置。

### 操作流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  确认卸载   │ -> │ 执行卸载    │ -> │  清理完成   │
└─────────────┘    └─────────────┘    └─────────────┘
```

### 详细操作

1. **确认卸载**：显示警告信息，确认用户意图
2. **执行卸载**：
   - 停止 OpenClaw 服务
   - 卸载 OpenClaw 包：`npm uninstall -g openclaw`
   - 删除配置文件：`~/.openclaw/`
   - 清理相关数据
3. **完成提示**：显示卸载结果

## 常见问题

### Q: 安装过程中卡住怎么办？

A: 
1. 查看右侧安装日志，定位具体步骤
2. 如果是下载步骤，尝试使用手动下载
3. 如果是安装步骤，检查网络连接
4. 重启应用后重试

### Q: 如何切换 AI 模型？

A:
1. 进入"AI配置"页面
2. 选择新的模型
3. 获取对应模型的 API Key
4. 保存配置

### Q: 飞书机器人不回复消息？

A:
1. 使用"一键诊断"检查渠道配置
2. 确认飞书应用已发布
3. 检查回调地址配置
4. 重启 OpenClaw 服务

### Q: 如何备份配置？

A:
1. 打开"设置"页面
2. 查看当前配置路径
3. 手动备份 `openclaw.json` 文件
4. 或在设置中导出配置
